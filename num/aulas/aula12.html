Aula 12: Solução de sistemas lineares: métodos iterativos.

Complexidade e métodos iterativos

A complexidade de um algoritmo mede como a quantidade de operações cresce quando o tamanho do problema aumenta. Para um sistema linear com n incógnitas, o método exato de Gauss-Jordan exige uma quantidade de operações proporcional a n3. Escrevemos isso como

$O\left(n^3\right).$

Isso significa que, se dobrarmos o número de incógnitas, o custo pode crescer aproximadamente oito vezes. Para sistemas muito grandes, esse custo pode ser alto.

Um método iterativo não tenta resolver o sistema de uma vez. Ele começa com uma aproximação inicial e melhora essa aproximação passo a passo. Cada iteração pode ter custo muito menor: para uma matriz cheia, normalmente o custo de uma iteração é da ordem de O(n2); para uma matriz esparsa, isto é, com muitos zeros, o custo pode ser proporcional apenas ao número de elementos não nulos.

Assim, se poucas iterações forem suficientes para atingir a precisão desejada, um método iterativo pode ter complexidade menor do que um método exato. Essa vantagem é especialmente importante em sistemas grandes e esparsos, que aparecem com frequência em simulações físicas, discretização de equações diferenciais e problemas de otimização.

Resolução do sistema linear

Função de iteração

Consideremos um sistema linear quadrado

$AX=B,$

onde A é uma matriz n×n, X é a matriz coluna das incógnitas e B é a matriz coluna do lado direito. Os métodos iterativos começam com uma aproximação inicial X(0) e produzem novas aproximações

$X^{\left(k+1\right)}=\Phi\left(X^{\left(k\right)}\right),$

onde Φ é a função de iteração para a coluna de incógnitas. Se a sequência converge, seu limite deve satisfazer o sistema original. A vantagem dos métodos iterativos é que cada passo pode ser simples e barato, especialmente quando a matriz A é grande e contém muitos zeros.

Critério de parada

Para uma única variável, um critério de parada comum é comparar duas aproximações consecutivas. Se xk e xk+1 estão muito próximos, paramos o processo:

$|x_{k+1}−x_k|<\varepsilon.$

Para um sistema linear, a aproximação não é um número, mas uma matriz coluna X. Portanto, a diferença entre duas aproximações também é uma matriz coluna:

$X^{\left(k+1\right)}−X^{\left(k\right)}.$

Podemos interpretar essa coluna como um vetor em um espaço com norma. Uma norma transforma um vetor em um número não negativo e permite definir uma distância:

$d\left(X,Y\right)=\|X−Y\|.$

Para ser uma norma, uma função deve satisfazer os seguintes axiomas:

  1. Positividade: $\|X\|\geq 0$, e $\|X\|=0$ somente quando X=0.
  2. Homogeneidade: $\|\alpha X\|=|\alpha|\,\|X\|$ para qualquer número α.
  3. Desigualdade triangular: $\|X+Y\|\leq \|X\|+\|Y\|$.

Assim, o critério de parada para métodos iterativos aplicados a sistemas lineares pode ser escrito como

$\|X^{\left(k+1\right)}−X^{\left(k\right)}\|<\varepsilon.$

Existem várias escolhas possíveis de norma. A norma cartesiana, também chamada norma euclidiana, é

$\|X\|_2=\sqrt{x_1^2+x_2^2+\cdots+x_n^2}.$

Mais geralmente, as p-normas são definidas por

$\|X\|_p=\left(|x_1|^p+|x_2|^p+\cdots+|x_n|^p\right)^{1/p},$ $\qquad p\geq 1.$

Dois casos importantes são a norma da soma e a norma do máximo:

$\|X\|_1=|x_1|+|x_2|+\cdots+|x_n|,$
$\|X\|_\infty=\max_i |x_i|.$

Também podemos usar normas ponderadas, quando algumas componentes devem ter mais peso do que outras. Se wi>0, uma p-norma ponderada pode ser escrita como

$\|X\|_{p,w}=\left(w_1|x_1|^p+w_2|x_2|^p+\cdots+w_n|x_n|^p\right)^{1/p},$ $\quad p\geq 1.$

A escolha da norma depende do problema. Em aplicações numéricas, a norma do máximo é muito usada porque é simples de calcular e controla o maior erro entre todas as componentes.

Separação da matriz

Vamos decompor a matriz A em três partes:

$A=D+L+U,$

onde D contém apenas os elementos da diagonal principal, L contém os elementos abaixo da diagonal, e U contém os elementos acima da diagonal:

$D=\left(\begin{array}{cccc} a_{11}&0&\cdots&0\\ 0&a_{22}&\cdots&0\\ \vdots&\vdots&\ddots&\vdots\\ 0&0&\cdots&a_{nn} \end{array}\right),$ $\qquad L=\left(\begin{array}{ccccc} 0&0&0&\cdots&0\\ a_{21}&0&0&\cdots&0\\ a_{31}&a_{32}&0&\cdots&0\\ \vdots&\vdots&\vdots&\ddots&\vdots\\ a_{n1}&a_{n2}&a_{n3}&\cdots&0 \end{array}\right),$ $\qquad U=\left(\begin{array}{ccccc} 0&a_{12}&a_{13}&\cdots&a_{1n}\\ 0&0&a_{23}&\cdots&a_{2n}\\ 0&0&0&\cdots&a_{3n}\\ \vdots&\vdots&\vdots&\ddots&\vdots\\ 0&0&0&\cdots&0 \end{array}\right).$

Assim, o sistema AX=B pode ser escrito como

$(D+L+U)X=B.$

Método de Jacobi

No método de Jacobi, isolamos a parte diagonal:

$DX=B−(L+U)X.$

A partir de uma aproximação X(k), calculamos a próxima aproximação por

$DX^{\left(k+1\right)}=B−(L+U)X^{\left(k\right)}.$

Como D é diagonal, esse sistema é muito simples de resolver. Para cada componente, obtemos

$x_i^{\left(k+1\right)}=\displaystyle{\frac{1}{a_{ii}}}\left(b_i−\sum_{j\neq i}a_{ij}x_j^{\left(k\right)}\right),$ $\quad i=1,2,\ldots,n.$

Observe que todos os valores do passo k+1 são calculados usando apenas os valores antigos do passo k.

Teorema de convergência do método de Jacobi. Suponhamos que A é estritamente diagonalmente dominante por linhas, isto é,
$\displaystyle{\frac{1}{|a_{ii}|}\sum_{j\neq i}|a_{ij}|}<1,\qquad i=1,2,\ldots,n.$
Então o método de Jacobi converge para a solução do sistema AX=B para qualquer aproximação inicial X(0).
Demonstração. Seja X a solução exata do sistema e seja E(k)=X(k)−X o erro da k-ésima aproximação. Subtraindo a fórmula exata da fórmula iterativa, obtemos
$e_i^{\left(k+1\right)}=−\displaystyle{\frac{1}{a_{ii}}\sum_{j\neq i}a_{ij}e_j^{\left(k\right)}}.$
Tomando valores absolutos,
$|e_i^{\left(k+1\right)}|\leq \displaystyle{\frac{1}{|a_{ii}|}\sum_{j\neq i}|a_{ij}|}\max_j |e_j^{\left(k\right)}|.$
Como A é estritamente diagonalmente dominante, existe q<1 tal que
$\displaystyle{\frac{1}{|a_{ii}|}\sum_{j\neq i}|a_{ij}|}\leq q<1,\qquad i=1,2,\ldots,n.$
Logo,
$\max_i |e_i^{\left(k+1\right)}|\leq q\max_i |e_i^{\left(k\right)}|.$
Portanto, o erro diminui pelo menos por um fator q a cada iteração. Assim, E(k) tende a zero, e X(k) converge para X. ∎

Método de Gauss-Seidel

No método de Gauss-Seidel, aproveitamos imediatamente os valores novos assim que eles são calculados. Escrevemos

$(D+L)X=B−UX.$

Então a iteração é dada por

$(D+L)X^{\left(k+1\right)}=B−UX^{\left(k\right)}.$

Como D+L é triangular inferior, os componentes de X(k+1) são calculados em ordem, de cima para baixo. Na forma componente, temos

$x_i^{\left(k+1\right)}=\displaystyle{\frac{1}{a_{ii}}}\left(b_i−\sum_{j=1}^{i−1}a_{ij}x_j^{\left(k+1\right)}−\sum_{j=i+1}^{n}a_{ij}x_j^{\left(k\right)}\right).$

A diferença em relação ao método de Jacobi é que, quando calculamos xi(k+1), os valores novos x1(k+1), ..., xi−1(k+1) já são usados. Por isso, Gauss-Seidel costuma convergir mais rapidamente do que Jacobi, quando ambos convergem.

Critérios de convergência

Para formular o critério de convergência, calculamos os números βi linha por linha:

$\beta_i=\displaystyle{\frac{1}{|a_{ii}|}\left(\sum_{j=1}^{i−1}|a_{ij}|\beta_j+\sum_{j=i+1}^{n}|a_{ij}|\right)},\quad i=1,2,\ldots,n.$

Como βi usa os valores β1, ..., βi−1 já calculados, esse critério reflete a própria lógica do método de Gauss-Seidel, que aproveita imediatamente os valores novos.

Critério de Sassenfeld. Se
$\max_i \beta_i<1,$
então o método de Gauss-Seidel converge para qualquer aproximação inicial.

Outro critério importante vale para o método de Gauss-Seidel:

Se A é simétrica e positivamente definida, então o método de Gauss-Seidel converge para qualquer aproximação inicial.

Esse critério é muito útil porque matrizes simétricas positivamente definidas aparecem com frequência em problemas de mínimos quadrados, energia, elasticidade, difusão e discretização de equações diferenciais.

Nos casos em que a convergência do método de Jacobi é garantida pelo critério da diagonal dominante, o método de Gauss-Seidel também converge. Assim, dentro desse critério, Gauss-Seidel converge sempre que Jacobi converge.

Em muitos problemas, Gauss-Seidel converge mais rapidamente do que Jacobi, pois usa imediatamente os valores novos. No entanto, isso não significa que Jacobi possa ser sempre descartado: Jacobi é mais simples de paralelizar, enquanto Gauss-Seidel depende da ordem em que as componentes são calculadas.

Resumo

  1. O método de Jacobi calcula todos os valores novos usando apenas os valores da iteração anterior.
  2. O método de Gauss-Seidel usa imediatamente os valores novos já calculados na mesma iteração.
  3. O método de Gauss-Seidel converge sempre que o de Jacobi converge.
