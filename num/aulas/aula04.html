Aula 4: Propagação de erros e instabilidade numérica.

Propagação de erros

Além do erro inicial nos dados e erro de aproximação na aritmética de ponto flutuante cada operação gera ainda o erro de arredondamento. Todos estes erros se acumulam. Por isso, normalmente o erro do resultado torna-se maior. Até operações muito simples podem dar o resultado não esperado, por exemplo, quando as frações decimais finitas correspondem às frações infinitas na base binária.

Em particular, a subtração dos números próximos é uma operação que pode levar ao crescimento rápido do erro relativo.

Podemos observar que operações matematicamente equivalentes podem gerar erros diferentes. Por isso, vale a pena utilizar operações que não levam ao crescimento rápido do erro. É claro que temos que evitar a subtração dos números próximos sempre quando for possível.

Às vezes, podemos achar uma fórmula alternativa para uma expressão,

$\displaystyle{\sqrt{10001}−100=(\sqrt{10001}−100)\times\frac{\sqrt{10001}+100}{\sqrt{10001}+100}}$ $=\displaystyle{\frac{(\sqrt{10001}){}^2−100^2}{\sqrt{10001}+100}=\frac{1}{\sqrt{10001}+100}}.$

A última expressão, sendo matematicamente equivalente à expressão inicial, não tem problema com crescimento do erro por causa de subtração. Por isso, ela tem que ser usada nas contas numéricas em vez da inicial.

Raízes de uma equação quadrática

O problema de subtração de números próximos aparece se resolvemos uma equação quadrática com coeficientes aproximados de a, b, e c:

$ax^2+bx+c=0.$

A fórmula de Bhaskara,

$\displaystyle{x_1=\frac{−b+\sqrt{b^2−4ac}}{2a}}, \qquad \displaystyle{x_2=\frac{−b−\sqrt{b^2−4ac}}{2a}},$

para b<0 aplica-se para calcular x1, enquanto para x2 pode levar ao crescimento do erro relativo, enquanto ∣ac∣≪b2.

Para b>0 o numerador de x2 contém uma soma de dois números negativos, enquanto para x1 podemos obter o crescimento do erro devido à subtração.

Por esta razão, a fórmula de Bhaskara podemos utilizar somente para calcular uma das raízes, enquanto a outra calculamos pela relação:

$\displaystyle{x_1x_2=\frac{c}{a}}.$

Instabilidade numérica

O erro inicial nos dados (por exemplo, erro de representação dos números) pode levar ao crescimento do erro de resultado das contas. Quando a acumulação de erros nas contas é rápida, o algoritmo é chamado numericamente instável. O resultado final pode ter um erro muito grande, em particular, maior que 100%. Tal crescimento catastrófico de erro significa que os algoritmos numericamente instáveis são praticamente inaplicáveis.

Quando os erros dos dados iniciais e contas intermediárias tem um efeito desprezível no resultado final, tal algoritmo é chamado numericamente estável. Somente os algoritmos estáveis são usados para obter os resultados úteis.

Consideremos as integrais

$\displaystyle{I_n=\int_0^1\frac{x^n}{x+5}dx}.$

Todas as integrais deste tipo são positivas. Para n=0, obtemos

$\displaystyle{I_0=\int_0^1\frac{dx}{x+5}=\ln{(x+5)}│_0^1=\ln{~\frac{6}{5}}}.$

Podemos achar também uma relação de recorrência,

$I_n=\displaystyle{\int_0^1\frac{x^n+5x^{n-1}−5x^{n-1}}{x+5}dx}$ $=\displaystyle{\int_0^1\left(\frac{x^{n-1}(x+5)}{x+5}−5\frac{x^{n-1}}{x+5}\right)dx}$ $=\displaystyle{\int_0^1x^{n-1}dx−5\int_0^1\frac{x^{n-1}}{x+5}dx}=\frac{1}{n}−5I_{n-1}.$

Comparamos valores das integrais com os calculados pela relação de recorrência.

Podemos observar que o erro absoluto cresce rápido e já para n=23 é maior que o valor aproximado, que significa que o erro absoluto é maior que 100%. Isto é claro pois obtemos o valor negativo como uma aproximação para integral positiva de I23.

Se o erro da aproximação inicial para I0 for maior que o erro de representação, vamos obter o resultado absurdo para menor índice de n. Por exemplo, se usar aproximação inicial 0.182 (3 casas decimais), vamos obter o valor negativo já para aproximação de I4.

Podemos concluir que esta recorrência, sendo aplicada para calcular as integrais a partir de aproximação para I0 é numericamente instável. Ao mesmo tempo, a aplicação da fórmula invertida dá um algoritmo numericamente estável.

$\displaystyle{I_{n-1}=\frac{1}{5}\left(\frac{1}{n}−I_{n}}\right).$

Podemos aproximar I20≈0 e achar o valor aproximado para I0.

Resumo

  1. Erro nos dados iniciais e erro de representação dos números na aritmética de ponto flutuante podem levar ao crescimento rápido de erro relativo do resultado.
  2. Operações que resultam no crescimento do erro, como, por exemplo, subtração de números próximos temos que evitar sempre quando for possível.
  3. Acumulação rápida dos erros num algoritmo significa que tal algoritmo não é praticamente útil. Este fenômeno é chamado de instabilidade numérica.
